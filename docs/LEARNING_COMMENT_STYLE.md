# Learning Comment Style

> Тег: АКТУАЛЬНО | Обновлён: 2026-03-12 | Версия: 1.0

## Зачем это нужно

Этот файл задает единый протокол, как ты помечаешь непонимание в коде и документации, а AI отвечает прямо рядом с местом вопроса.

## Быстрый цикл работы

1. Ты ставишь тег рядом с местом, где вопрос.
2. AI находит теги поиском по репозиторию.
3. AI отвечает рядом через ответный тег.
4. После того как вопрос закрыт, тег можно удалить или перенести в архив.

## Теги владельца (твои)

Используй эти теги в Scala и Markdown.

- QUESTION(U): общий вопрос по месту в коде.
- WHY(U): почему решение сделано именно так.
- CONFUSED(U): не понимаю поток/контекст/типы.
- TRACE(U): хочу пошаговый разбор потока данных.
- GLOSSARY(U): непонятный термин или аббревиатура.
- CHECK(U): проверь мою гипотезу или понимание.

Пример для Scala:

// QUESTION(U): почему здесь именно zipPar, а не последовательный publish?
// TRACE(U): покажи путь пакета от channelRead до Kafka

Пример для Markdown:

<!-- QUESTION(U): почему этот топик имеет такой partition key? -->
<!-- CONFUSED(U): не понимаю разницу между gps-events и gps-events-rules -->

## Теги ответов AI

- ANSWER(AI): короткий прямой ответ на вопрос.
- WALKTHROUGH(AI): пошаговый разбор с привязкой к месту.
- GLOSSARY(AI): определение термина простыми словами.
- NEXTSTEP(AI): что проверить дальше локально.

Пример:

// ANSWER(AI): zipPar здесь безопасен, потому что публикации независимы.
// WALKTHROUGH(AI): 1) channelRead принимает ByteBuf 2) parser.parseData 3) publish в gps-events...
// NEXTSTEP(AI): проверь тестом падение одного из publish-эффектов и ожидаемое поведение.

## Шаблон ответа AI для обучения

Для каждого сложного ответа AI использует 4 блока:

1. Что это делает
2. Почему так
3. Как проверить
4. Что может сломаться

## Правила качества для учебных ответов

- Ответ в том же файле и максимально рядом с вопросом.
- Коротко и предметно: 3-8 строк на ANSWER(AI).
- Если нужен длинный разбор, использовать WALKTHROUGH(AI) и дать шаги проверки.
- Для важных мест добавлять NEXTSTEP(AI) с конкретной командой или тестом.

## Поиск всех учебных пометок

Рекомендуемый поиск:

rg -n "QUESTION\(U\)|WHY\(U\)|CONFUSED\(U\)|TRACE\(U\)|GLOSSARY\(U\)|CHECK\(U\)|ANSWER\(AI\)|WALKTHROUGH\(AI\)|GLOSSARY\(AI\)|NEXTSTEP\(AI\)" .

## Мини-словарь уровней непонимания

- LEVEL 1: термин неясен -> ставь GLOSSARY(U)
- LEVEL 2: место неясно -> ставь QUESTION(U)
- LEVEL 3: поток неясен -> ставь TRACE(U)
- LEVEL 4: решение неочевидно -> ставь WHY(U)

## Практика для глубокой учебы по сервису

Для каждого нового модуля делай минимум 3 пометки:

- QUESTION(U): про главную ответственность файла
- TRACE(U): про главный поток данных
- WHY(U): про одно нетривиальное решение

Это позволяет быстро накапливать точечную карту понимания сервиса.
